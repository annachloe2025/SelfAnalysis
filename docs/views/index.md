# ビュー層 — 多面的な「私」の見え方

ビュー層は、データ層の atom を**特定の視点で束ね直した読み物**である。
本文を持たず、データ層への参照と視点別のナラティブで構成される。

## ビュー層の運用原則

- ビューは**データ層への参照と視点別解説**でできている。本文を持たない
- 同じ atom が**複数のビューに登場してよい**。多面的に見られることが目的
- ビューには `view_type / audience / reading_time / last_updated` を必ず明示
- 「強み」「特技」の framing を避け、**生得的にこの体質ゆえに現状こうなった** 型の記述を徹底

詳細は [_design/04_ビュー層カタログ.md](../../_design/04_ビュー層カタログ.md) と [_design/07_運用ルール.md](../../_design/07_運用ルール.md) §4.7 を参照。
